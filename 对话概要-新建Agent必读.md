## 项目一：纪律 App（Flutter 任务管理应用）

**位置**：`D:\03_Dev\Projects\app_dev\Plan\`

### 产品定位
AI 对话式任务管理 + 心情追踪。核心用户是「懒、没有目标的人」，帮他们把模糊的待办事项变成清晰可执行的行动。忙碌的高效能人士也能用，但产品不做复杂化。

### 技术栈
- Flutter + Dart
- Riverpod（状态管理）
- GoRouter（路由，StatefulShellRoute 底部导航）
- Supabase（后端，双模式：有配置用云端，无配置自动切本地模拟数据）
- OpenClaw Gateway（DeepSeek API 代理，SSE 流式响应）
- SharedPreferences（本地持久化）

### 当前状态
- `flutter analyze`：0 issues
- `flutter test`：101/101 passed
- 代码在 `app/` 子目录下，所有命令从 `app/` 执行

### 已实现功能
| 模块 | 内容 |
|------|------|
| 认证 | 邮箱注册/登录，auth guard，状态机（unauthenticated → loading → authenticated → error） |
| 任务管理 | CRUD、状态筛选（待办/进行中/已完成）、优先级（高中低）、任务分类（未分类/学习/工作/生活/健康/社交/创作/财务） |
| AI 对话 | SSE 流式响应、欢迎消息、空状态引导建议、下拉刷新 |
| 心情记录 | great/good/okay/bad/terrible 五档 + 备注、历史列表、下拉刷新 |
| 统计仪表盘 | 完成率环形图、今日活动、分类分布进度条、状态分布 |
| 引导页 | 3 页 onboarding，可跳过，SharedPreferences 持久化已读状态 |
| 邀请系统 | 邀请码生成/分享，邀请 3 人 = 1 个月 Pro |
| 深色主题 | 全局暗色视觉风格 |
| 模式选择 | trial/push/peace 三种任务模式 + 名人名言展示 |

### 架构分层（依赖方向单向）
```
models / config → services → providers → pages / widgets → router
```
- Service 层永不 catch 异常，全部冒泡到 provider 转 `AsyncValue.error()`
- 每个页面必须处理 data / loading / error 三种状态
- Provider 命名：`camelCase + Provider` 后缀

### 待办
- 无。功能实现已完整，analyze 和 test 均干净。
